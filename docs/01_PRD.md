# 제품 요구사항 정의서 (PRD)

**프로젝트명:** Slack 기반 지출품의서 자동화 에이전트 (Expense Report Agent)

**목표:** 개인 카드 영수증 처리를 자동화하여 구성원의 행정 업무 시간 단축 및 정산 정확도 향상

---

## 1. 핵심 기능 (Core Features)

### 1.1 영수증 수집
- Slack 채널에서 봇 멘션(`@Agent`)과 함께 업로드된 이미지 수집
- 지원 이미지 포맷: JPG, PNG, HEIC 등

### 1.2 이미지 분석 (Vision AI)
영수증에서 다음 정보를 자동 추출:
- 상호명 (Merchant Name)
- 거래 일시 (Transaction Date)
- 금액 정보 (공급가액 / 부가세)
- 품목 (Items)

### 1.3 Google Sheets 연동
- **시트 생성:** 마스터 템플릿(`양식_지출품의서_이름`)을 복사하여 새로운 시트 생성
- **데이터 입력:** 추출된 데이터를 `지출결의서_템플릿` 시트의 지정된 셀에 자동 입력
- **이미지 첨부:** 영수증 원본 이미지를 `영수증 첨부` 시트에 삽입

### 1.4 휴먼 인터랙션
- **검토 요청:** 작성 완료 후 사용자에게 DM 또는 스레드로 검토 요청
- **확인 프로세스:** 사용자 "확인 완료" 트리거 시 최종 처리 담당자에게 알림 발송

---

## 2. 사용자 워크플로우 (User Journey)

### Step 1: 요청
- 사용자가 프로젝트 채널(예: `#pj-foodcare`)에서 영수증 사진 업로드
- 봇을 태그하여 처리 요청: `@Agent`

### Step 2: 처리
- Agent가 즉시 응답: "지출품의서를 작성 중입니다..."
- 백그라운드에서 다음 작업 수행:
  - 영수증 이미지 다운로드
  - AI 분석을 통한 데이터 추출
  - Google Sheets 생성 및 데이터 입력

### Step 3: 검토
- Agent가 작성된 Google Sheets 링크를 답글로 전달
- **메시지 예시:**
  ```
  우디님, 지출품의서 초안이 작성되었습니다.
  내용을 확인하고 수정이 필요 없으면 '완료'라고 답글을 달아주세요.

  📄 [Google Sheets 링크]
  ```

### Step 4: 확정
- 사용자가 시트 내용을 확인
- 이상이 없으면 "완료" 응답 (또는 버튼 클릭)

### Step 5: 통보
- Agent가 지정된 재무 담당자에게 최종 메시지 발송

## 3. 비기능 요구사항 (Non-Functional Requirements)

### 3.1 성능
- 영수증 처리 시간: 30초 이내
- 동시 처리 가능 건수: 최소 5건

### 3.2 정확도
- 영수증 데이터 추출 정확도: 95% 이상
- 금액 계산 오차: 0원 (정확한 계산 필수)

### 3.3 보안
- Slack Token, Google API 키 등 민감 정보는 환경 변수로 관리
- 영수증 이미지는 처리 후 임시 저장소에서 삭제

### 3.4 확장성
- 향후 다른 프로젝트 채널 추가 가능하도록 설계
- 템플릿 형식 변경에 유연하게 대응 가능한 구조

---

## 4. 제약 사항 (Constraints)

### 4.1 기술적 제약
- Slack Workspace 내에서만 작동
- Google Workspace 계정 필요
- Claude API 사용량 제한 고려

### 4.2 비즈니스 제약
- 한국어 영수증에 최적화
- 개인카드 사용 내역만 처리 (법인카드는 별도 프로세스)

---

## 5. 성공 지표 (Success Metrics)

- 지출품의서 작성 시간 단축: 기존 대비 80% 이상 감소
- 데이터 입력 오류율: 5% 이하
- 사용자 만족도: 5점 만점에 4.0 이상
- 월간 처리 건수: 최소 50건
